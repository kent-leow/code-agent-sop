# Requirements-to-Code Workflow

An agent-driven pipeline for transforming raw business requirements into production-ready code with full traceability.

## Overview

```
Raw Requirements
  → @generate-plan   →  plan.md  (re-run to refine)
  → @generate-task   →  task-001.md, task-002.md, ...  (re-run to refine)
  → @execute-task    →  production code + tests
  → @fix-task        →  fix.md  (post-implementation fixes)
```

Each stage produces a concrete artefact in `.docs/<task-name>/`. Nothing moves forward until the previous output is complete and verified.

## File Structure

```
.docs/
└── <task-name>/              # One folder per feature / task
    ├── plan.md               # Stage 1 output: requirements, AC, open questions
    ├── jira.json             # Jira story + sub-task keys
    ├── task-001.md           # Stage 2 output: vertical slice 1
    ├── task-002.md           # Stage 2 output: vertical slice 2
    ├── fix.md                # Stage 4 output: post-implementation fix log
    └── figma/                # Cached Figma assets (auto-created by agents)
        ├── <nodeId>.json
        ├── <nodeId>.png
        └── <nodeId>.md
```

## Stages

### Stage 1 — Plan
**Agent**: `@generate-plan`
- **Input**: Raw requirement, feature request, or user description
- **Output**: `.docs/<task-name>/plan.md` + Jira story (via `jira-ticket` skill)
- **plan.md sections**: Summary, Scope, Acceptance Criteria, Open Questions, Changelog
- **Refine**: re-run with `plan.md` path + changes — agent auto-detects and updates in place

### Stage 2 — Generate Tasks
**Agent**: `@generate-task`
- **Input**: `.docs/<task-name>/plan.md` (no unresolved blocking questions)
- **Output**: `task-001.md`, `task-002.md`, … (one per vertical slice)
- Each file contains: task list, file-level checklist, code guidance, test requirements, Done When criteria
- **Refine**: re-run with `task-NNN.md` path + corrections — agent auto-detects and updates in place

### Stage 3 — Execute
**Agent**: `@execute-task`
- **Input**: `task-NNN.md` path
- **Output**: Production code, tests, all checkboxes marked `[x]`
- Checks prerequisites (earlier slices must be Done) before starting
- Syncs sibling task files for cross-slice impact
- Creates Jira sub-task and logs it in `jira.json`

### Stage 4 — Fix (post-implementation)
**Agent**: `@fix-task`
- **Input**: folder path + issue description(s) (review comments, bug reports, failing tests)
- **Output**: `fix.md` created/updated; all fixes applied; task/plan docs updated
- `fix.md` items are numbered `FIX-001`, `FIX-002`, … with source and affected files

## Supporting Agents

| Agent | When to use |
|---|---|
| `@investigate` | Root-cause analysis, ad-hoc debugging, "why did X happen" queries |
| `@gitlab-mr` | Review, summarise, or fix comments on a GitLab MR |
| `@snapshot-sync` | Create or refresh `SNAPSHOT.md` in any repo |

## Naming Conventions

| Artefact | Pattern | Example |
|---|---|---|
| Task folder | `kebab-case` | `create-form-and-application` |
| Plan | `plan.md` | `.docs/create-form-and-application/plan.md` |
| Task slice | `task-NNN.md` | `task-001.md` |
| Fix log | `fix.md` | `.docs/<task>/fix.md` |
| Jira state | `jira.json` | `.docs/<task>/jira.json` |

## Integrations

### Jira
- `@generate-plan` auto-creates a Jira story after writing `plan.md`
- `@execute-task` auto-creates a Jira sub-task per slice
- All keys stored in `jira.json` for traceability
- Requires: `JIRA_TOKEN`, `JIRA_EMAIL`, `JIRA_BASE_URL`, `JIRA_PROJECT_KEY` in `~/.zshenv`

### Figma
- Any agent that encounters a Figma URL fetches and caches design context under `figma/`
- Cache-first: existing `<nodeId>.json` is reused unless a refresh is signalled
- Requires: `FIGMA_TOKEN` in `~/.zshenv` (fallback when Figma MCP is unavailable)

## Quality Gates

- **Traceability**: every task-NNN.md task links back to an AC row in `plan.md`
- **Prerequisites**: `@execute-task` refuses to start if earlier slices have open Done When items
- **Test coverage**: every logic task must have a test child task
- **Changelog**: every refinement appends a dated entry to the affected file
