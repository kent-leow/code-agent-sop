# Requirements-to-Code Workflow

Agent-driven pipeline: raw requirements ‚Üí production code with full traceability.

## Pipeline

```
Raw Requirements
  ‚Üí @generate-plan   ‚Üí  plan.md  (re-run to refine)
  ‚Üí @generate-task   ‚Üí  task-001.md, task-002.md, ...
  ‚Üí @execute-task    ‚Üí  production code + tests
  ‚Üí @fix-task        ‚Üí  fix-{datetime}.md  (per issue batch)
```

Each stage produces artefacts in `.docs/<feature>/`. Nothing advances until prior output is verified.

## Stages

| Stage | Agent | Input | Output |
|---|---|---|---|
| 1. Plan | `@generate-plan` | Raw requirement | `plan.md` + Jira story |
| 2. Tasks | `@generate-task` | `plan.md` (no blocking questions) | `task-NNN.md` per slice |
| 3. Execute | `@execute-task` | `task-NNN.md` path | Code + tests + checkboxes marked |
| 4. Fix | `@fix-task` | folder + issues | `fix-{datetime}.md` + fixes applied |

## Supporting Agents

| Agent | Use case |
|---|---|
| `@investigate` | Root-cause analysis, ad-hoc debugging |
| `@git-review` | Review/summarise/fix comments on PR/MR |
| `@snapshot-sync` | Create/refresh `SNAPSHOT.md` |

## Naming Conventions

| Artefact | Pattern | Example |
|---|---|---|
| Feature folder | `kebab-case` | `create-form-and-application` |
| Plan | `plan.md` | `.docs/create-form/plan.md` |
| Task slice | `task-NNN.md` | `task-001.md` |
| Fix log | `fix-{datetime}.md` | `fix-20260424-143000.md` |
| Jira state | `jira.json` | `.docs/<feature>/jira.json` |

## Integrations

| System | Behaviour |
|---|---|
| Jira | `generate-plan` creates story; `execute-task` creates sub-task; keys in `jira.json` |
| Figma | `generate-plan` creates `figma/` folder under plan folder; stores context.json, screenshot.png, summary.md, flow.json; reuses cached files on hit |

Requires: `JIRA_TOKEN`, `JIRA_EMAIL`, `JIRA_BASE_URL`, `JIRA_PROJECT_KEY`, `FIGMA_TOKEN` in `~/.zshenv`.

## Quality Gates

- Traceability: every task links to AC in `plan.md`
- Prerequisites: `execute-task` refuses if earlier slices have open items
- Test coverage: every logic task has a test child
- Changelog: every refinement appends dated entry
